## Context

目前，系统在 C108 社团的 IP 标记上依赖一维的滑动窗口空间传播机制。该机制以社团在单排内的相对摊位号（数字）为数轴进行排序，并要求前后距离不超过 5 个展位。这一简化的拓扑设计使得系统无法计算过道对面（面对面）、排后方（背靠背）的物理邻域，且在遇到连续空缺摊位时极易断流，同时不支持多标签概率分布。为此，本设计提出构建二维展馆坐标几何模型，并在图网络上运行带衰减的标签传播算法（2D-LPA），同时以二维几何距离解算最优逛展路径。

## Goals / Non-Goals

**Goals:**
- **物理坐标数字化**：将类似 `日曜日 東馆 Ｈ12a` 的排位表示映射为标准的二维空间 $(X, Y)$ 米级笛卡尔坐标。
- **二维加权邻居建图**：在阈值 $d \le 3.5$ 米内自动连接近邻展位，建立带连接类型权重（左右邻、背靠背、跨过道）的拓扑图网络。
- **带衰减多标签 LPA 算法**：社团打标采用多维实数向量，允许进行带阻尼衰减的多轮标签扩散，提供 $p \ge 0.3$ 的多标签过滤标记。
- **二维最优逛展排线**：基于最近邻启发式路径解算，生成从起点（展馆入口）出发、遍历所有目标摊位的最短距离路线推荐。

**Non-Goals:**
- **三维模型渲染与动画**：不进行任何三维可视化呈现，只进行后台的二维向量几何计算。
- **实时的场馆地图绘制与导航**：路线解算只输出排序后的摊位列表，不提供前端地图 UI 渲染。

## Decisions

### 1. 展馆排位配置网格设计 (Grid Config Strategy)
* **方案选择**：为了避免复杂的 CAD 图解，我们在项目中引入静态布局网格配置。
* **设计细节**：
  - 在配置或 `src/spatial_mapper.py` 中，将每一个 Hall 中的每个 Block（如 `Ｈ` 街区）定义为一个基准轴线。例如：在东馆，`Ｈ` 街区在 $X = 35.0$ 米处，空间号沿 $Y$ 从 $0$ 增加到 $40$ 米。
  - `a` 面和 `b` 面展位沿 $X$ 轴左右偏移 $-0.5$ 和 $+0.5$ 米，使其天然具备“背靠背”的几何接近性。
* **原因**：这能够在极小开发成本下提供非常精准的漫展物理邻域计算精度。

### 2. 标签传播与矩阵迭代计算 (LPA Vector Propagation)
* **方案选择**：选用带阻尼系数 $\alpha = 0.6$ 的半监督多分类标签传播算法（Label Propagation Algorithm）。
* **设计细节**：
  - 图中每个节点 $i$ 代表一个社团，用实数概率向量 $\mathbf{P}_i$ 记录其各 IP 概率。
  - 迭代更新方程：
    $$\mathbf{P}_i^{(t+1)} = \alpha \sum_{j \in \mathcal{N}(i)} \bar{W}_{ij} \mathbf{P}_j^{(t)} + (1 - \alpha) \mathbf{y}_i$$
    其中阻尼 $\alpha$ 能让火种能量（概率）向外衰减扩散，如第一代 $1.0 \times 0.6 = 0.6$，第二代 $0.6 \times 0.6 = 0.36$，到第四代低于 $0.3$ 则被自动截断，解决“断流”的同时避免了无边界的污染。
* **原因**：能够同时解决弱势 IP 的短距离蔓延（不需要在周围凑够两个以上相同种子），且能够计算“百货大杂烩”多标签占比。

### 3. 最近邻逛展解算器 (Nearest Neighbor TSP Solver)
* **方案选择**：采用贪心最近邻（Greedy Nearest Neighbor）算法作为旅行商问题（TSP）的启发式解算器。
* **设计细节**：
  - 设定东馆、西馆、南馆的几何大门入口 $(X_0, Y_0)$。
  - 从入口出发，寻找最近的已标定摊位，加入路线链条。
  - 接着以当前摊位为起点，在其余未访问摊位中寻找二维距离最近的一个，迭代完成全量规划。
* **原因**：通常逛展摊位 $N \le 500$，最近邻算法的复杂度为 $O(N^2)$，在 Python 中可在几毫秒内解算完毕，产生的路径在二维几何上基本无交叉折返，效果极佳。

## Risks / Trade-offs

* **[Risk]** Comiket 不同届（如 C108 到 C109）的展位网格可能微调。
  - **[Mitigation]** 将各街区坐标网格以 JSON 格式独立抽取为配置文件（如 `data/layout_grid_c108.json`），使坐标映射逻辑与解析引擎解耦。
* **[Risk]** 多 IP 概率传播中，极近邻摊位可能因为权重过高造成概率交叉污染。
  - **[Mitigation]** 对边权重 $W_{ij}$ 增加衰减约束，并设置较高的最终判定截断阈值（$Threshold = 0.3$），概率低于此值的 IP 一律抛弃。
