## Context

在 Comiket 展会中，社团的题材分类由 WebCatalog 提供。然而，官方给出的题材往往过于宏观（如“社交/网络游戏”包含了数百款不同的游戏）。
为了支持对热门具体 IP（如《明日方舟》、《原神》、《蔚蓝档案》等）在 Comiket 中的大盘占有率、空间莫兰指数以及引流漏斗进行细粒度对比研究，我们急需一套对社团二创 IP 的精确识别与标记机制。

目前很多社团的 WebCatalog 简介部分缺失或非常简短，只通过纯文本检索会遗漏大量实际在物理排摊上被排在一起的同 IP 社团。因此我们需要结合“文本匹配种子”、“商品数据反推”以及“物理空间邻域传播”这三种策略进行智能综合打标。

## Goals / Non-Goals

**Goals:**
- 设计多对多关系表 `circle_ip_tags`，以支持同一社团拥有多个 IP 标签。
- 创建数据库联表查询视图 `v_circles_with_ip_tags`，自动完成星期的星期几格式（土、日、火、水）与逻辑日期（1、2）的对齐转换。
- 实现一键打标命令行指令 `--tag-circles`。
- 实现命令行路线指引查询 `--search-ip <name>`，以 Day -> Hall -> Block -> Space 顺序展示结果。
- 在商品数据 CSV 导出（`--export-goods`）中包含 IP 标签字段。

**Non-Goals:**
- 不涉及 WebCatalog 页面数据的在线拉取更新（这属于 `--sync-circles` 职责）。
- 不破坏或修改已存在的物理表（如 `circles` 和 `c107_circles`）的 schema 结构，保证已有单元测试与分析脚本完全兼容。
- 本次打标暂不涉及基于大语言模型（LLM）对每一段简介做高成本的语义意图理解打标（仅使用正则表达式的多语言多词干关键词匹配）。

## Decisions

### 决策一：采用外键关系关联表 `circle_ip_tags` 而不是直接修改主表
- **理由**：
  - **多标签支持**：许多社团为多 IP 混合创作（如同时出明日方舟和蔚蓝档案的图），单列标记无法支撑。
  - **不污染物理表**：`circles` 和 `c107_circles` 是已存在且有大量依赖脚本引用的表。建立独立的关系表可以做到松耦合，非常易于回滚，且不影响已有表。
- **替代方案**：直接在 `circles` 和 `c107_circles` 中追加 `genre_code`, `sub_ip` 字段。缺点是单字段不支持多标签，且需要在各届社团表发生结构迁移。

### 决策二：物理空间邻域传播（Spatial Propagation）滑窗过滤机制
- **逻辑设计**：
  Comiket 物理布展是将画同一 IP 的作者们连续排布的。如果在某个 `(day, hall, block)` 分组内，发现有若干个摊位通过文本或商品直接匹配到了 IP（称为“种子点”），那么我们可以推导位于该种子区间中间的空白摊位同样属于这个 IP。
  - **计算方法**：
    1. 设一个排中某摊位的数字编码为 $x$ (如 "め 24a" -> 24)。
    2. 如果该摊位 $x$ 自身未匹配到 IP，但在其区间 $[x-5, x+5]$ 内存在至少 2 个已被确认为某 IP 标签的种子社团，且该排所有已知种子社团没有被标记为其他 IP，那么我们将该摊位自动打标为此 IP。
    3. 空间传播生成的标签置信度（Confidence）设为 `0.8`，标签来源标记为 `spatial`。

### 决策三：在 `main.py` 新增 `--search-ip` 便捷物理逛展路线排序
- **理由**：
  - 便于同人爱好者快速生成逛展路线。
  - **排序逻辑**：`Day (ASC)` -> `Hall (ASC)` -> `Block (ASC)` -> `Space (ASC)`。这样可以让摊位按照从展馆一端到另一端、从排头到排尾的顺序推荐，极大提高实用性。

## Risks / Trade-offs

- **[Risk] 空间传播可能发生边缘“误伤”**
  - **描述**：如果两个 IP 专区被官方靠在一起（例如め 09 是明日方舟，め 10 变成了原神），传播算法可能会把边缘的留空摊位误打标。
  - **Mitigation**：
    1. 限制空间传播仅适用于在 $[x-5, x+5]$ 范围内密度足够的区域；
    2. 将传播标签置信度标记为较弱的 `0.8`（种子为 `1.0`），且来源列设为 `spatial`。这可以让高级分析或精细的Moran's I空间自相关计算选择性地排除这些传播点。
